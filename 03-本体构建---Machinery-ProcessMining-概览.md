# Machinery (Process Mining) 概览

> 更新时间：2025-12-28
> 文档来源：https://palantir.com/docs/foundry/machinery/overview/

## 概述

**Machinery** 是一个用于理解和流程管理的应用程序，用于识别不需要的行为并改进以实现期望的结果。在 AIP 的 LLM 驱动功能的支持下，您可以将 Machinery 用作编排多个自动化的可靠框架。

## 核心价值

### 主要功能

| 功能 | 说明 |
|------|------|
| **流程理解** | 理解流程的各个方面 |
| **流程管理** | 管理整个流程生命周期 |
| **行为识别** | 识别不需要的行为 |
| **流程改进** | 实现持续改进 |
| **AIP 编排** | 编排多个 AI 代理 |

### 流程建模

许多现实世界的事件可以被视为流程，例如：

```
┌─────────────────────────────────────────────────────────────┐
│              现实世界流程示例                                │

│  ├── 审批流程                                              │
│  ├── 采购流程                                              │
│  └── 订单到收款                                            │
│                                                             │
│  政府运营                                                    │
│  ├── 许可审批                                              │
│  ├── 监管流程                                              │
│  └── 公共服务                                              │
│                                                             │
│  医疗程序                                                    │
```

在流程中，实体（如文档、设备或个人）会随时间经历状态变化。

## 支持的工作流

Machinery 支持以下工作流：

### 1. 流程优化

**功能**：利用 AIP 和自动化解决流程低效问题

- 识别瓶颈
- 自动化重复任务
- 优化流程路径

### 2. AIP 用例管理

**功能**：通过编排多个 AI 代理端到端管理 AIP 用例

- 编排 AI 代理
- 协调自动化任务
- 监控执行状态

### 3. 流程挖掘

**功能**：从外部事件日志挖掘正在进行的流程以获得对现有流程的可见性

- 分析事件日志
- 发现实际流程
- 识别偏差

### 4. 性能监控

**功能**：定义和监控性能指标和期望以识别流程瓶颈

- 设置 KPI
- 监控性能
- 识别问题

### 5. 运营应用

**功能**：构建运营应用程序以通过实时人工干预监督 AIP 工作流

- 实时监控
- 人工干预
- 决策支持

## 使用场景

### 1. 保险

**场景**：保险索赔流程

**流程**：
- 从最初提交到最终决策捕获保险索赔流程
- 相关文档的 OCR
- 半自动化处理
- 人工批准

### 2. 医院运营

**场景**：患者旅程建模

**流程**：
- 端到端建模患者护理旅程
- 通过医疗文档生成支持运营

### 3. 采购到付款

**场景**：采购流程优化

**流程**：
- 考虑从请购到付款的采购流程
- 提高工作效率

### 4. 订单到收款

**场景**：销售流程优化

**流程**：
- 优化从订单接收到收款的销售流程
- 提高客户满意度和现金流

## 核心概念

### 1. 流程图

**定义**：流程的可视化表示

**功能**：
- 显示状态、转换和操作的流动
- 可视化期望行为和偏差
- 支持交互式探索

### 2. 流程挖掘

**定义**：从事件日志中发现流程

**功能**：
- 从日志中挖掘正在进行的流程
- 在图表上可视化流程
- 识别瓶颈和偏差

### 3. 流程分析

**定义**：分析流程性能

**功能**：
- 定义和监控性能指标
- 识别瓶颈
- 发现改进机会

### 4. 流程改进

**定义**：实施流程变更

**功能**：
- 基于数据做出决策
- 实施变更
- 监控改进效果

## 主要功能

- 设置约束

**功能**：
- 连接到事件日志
- 连接到 Ontology 对象
- 集成实时数据

### 3. 流程挖掘

**功能**：
- 从事件日志挖掘流程
- 发现实际流程路径
- 识别偏差

**数据源选项**：
- 流程对象
- 日志对象
- 两者结合

### 4. 分析和监控

**功能**：
- 定义性能指标
- 设置期望
- 监控执行
- 识别瓶颈
