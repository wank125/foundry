# Upgrade Assistant 概览

> 更新时间：2025-12-28
> 文档来源：https://palantir.com/docs/foundry/upgrade-assistant/overview/

## 概述

**Upgrade Assistant（升级助手）** 提供对即将到来的平台变更的可见性。平台升级等变更在某些情况下可能会导致现有工作流出现问题。

为了帮助适应平台变更，Upgrade Assistant 提供了受特定平台升级影响的资源列表。每次升级还附带了有关如何修复受影响问题的说明。

为了确保采取行动，每个资源都分配给用户和组。Upgrade Assistant 根据各种规则和启发式方法确定资源分配者。

随着升级截止日期的临近，Upgrade Assistant 将向已分配资源的用户发送通知，提醒他们采取必要的操作以防止工作流中断。

## 核心功能

### 1. 平台变更可见性

**功能**：
- 查看即将到来的平台变更
- 了解升级对现有工作流的影响
- 获取修复指导

### 2. 资源影响识别

**功能**：
- 识别受平台升级影响的资源
- 提供详细的资源列表
- 显示影响级别和类型

### 3. 资源分配

**功能**：
- 自动分配资源给相关用户和组
- 基于规则和启发式方法
- 确保责任明确

### 4. 通知提醒

**功能**：
- 发送通知给已分配资源的用户
- 在截止日期临近时提醒
- 防止工作流中断

## 主要功能

### 1. 平台变更（Platform Changes）

**功能**：
- 显示所有即将到来的平台变更
- 提供变更详情和影响分析
- 按优先级排序

### 2. 资源识别（Identify Impacted Resources）

**功能**：
- 扫描所有平台资源
- 识别受影响的资源
- 按影响类型分类

### 3. 资源分配（Resource Assignment）

**分配规则**：
- 基于资源所有者
- 基于资源编辑者
- 基于资源使用模式
- 基于自定义规则

### 4. 通知管理（Notifications）

**通知类型**：
- 初始分配通知
- 截止日期提醒
- 升级即将执行通知
- 逾期警告

### 5. 忽略资源（Ignore Resources）

**功能**：
- 标记不相关资源
- 过滤误报
- 自定义忽略规则

### 6. 维护操作员（Maintenance Operators）

**功能**：
- 配置维护操作员
- 自动化修复流程
- 批量处理资源

## 资源类型

Upgrade Assistant 支持多种资源类型：

| 资源类型 | 说明 |
|---------|------|
| **Datasets** | 数据集 |
| **Pipelines** | 数据管道 |
| **Functions** | 函数 |
| **Workshop Modules** | Workshop 模块 |
| **Ontology Objects** | 本体对象 |
| **Actions** | 操作 |
| **Links** | 链接 |

## 使用场景

### 1. 模型弃用

**场景**：某个 LLM 模型即将被弃用。

**处理流程**：
1. Upgrade Assistant 识别所有使用该模型的资源
2. 将资源分配给相关用户
3. 提供迁移指导
4. 发送提醒通知

### 2. API 变更

**场景**：平台 API 发生重大变更。

**处理流程**：
1. 检测所有使用旧 API 的代码
2. 识别需要更新的函数
3. 提供迁移指南
4. 跟踪修复进度

### 3. 功能弃用

**场景**：某个功能即将被弃用。

**处理流程**：
1. 识别使用该功能的应用
2. 分配给应用所有者
3. 提供替代方案
4. 监控迁移进度

## 最佳实践

### 1. 定期检查

- **定期查看**：定期检查 Upgrade Assistant
- **及时响应**：及时响应通知
- **优先处理**：优先处理高影响资源

### 2. 资源管理

- **清晰所有权**：确保资源有明确的所有者
- **及时更新**：及时更新资源分配
- **文档记录**：记录修复过程

### 3. 测试验证

- **测试环境**：在测试环境中验证修复
- **充分测试**：充分测试工作流
- **备份计划**：准备回滚计划

### 4. 团队协作

- **沟通协调**：与团队沟通协调
- **分配责任**：明确分配责任
- **共享知识**：共享修复经验

## 界面导航

### 主界面

Upgrade Assistant 主界面包含：

- **平台变更列表**：显示所有即将到来的变更
- **受影响资源**：显示每个变更影响的资源
- **分配状态**：显示资源分配状态
- **进度跟踪**：显示修复进度

### 资源详情

每个资源详情页面包含：

- **资源信息**：资源名称、类型、位置
- **影响描述**：描述如何受影响
- **修复指导**：提供修复步骤
- **分配信息**：显示分配给谁

### 操作

可用操作：

- **分配资源**：手动分配资源
- **标记完成**：标记修复完成
- **忽略资源**：忽略不相关资源
- **添加注释**：添加修复注释

- **自定义通知**：自定义通知规则

## 限制

### 当前限制

- **自动修复**：不支持自动修复
- **跨注册**：不支持跨注册资源
- **历史数据**：不保留历史分配数据
