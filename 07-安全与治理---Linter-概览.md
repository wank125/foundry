# Linter 概览

> 来源：https://www.palantir.com/docs/foundry/linter/
> 分类：管理与赋能 / 代码质量

## 概述

**Linter** 应用程序检查 Foundry 的状态是否存在反模式，并建议将资源移至更好状态的方法。您可以使用这些建议来降低成本，并提高管道的稳定性和弹性。

![Linter 应用程序着陆页](https://www.palantir.com/docs/resources/foundry/linter/splash.png)

使用 Linter，您可以更好地了解 Foundry 提供的广泛功能，同时监控可能有助于您用例目标的任何平台更新。

任何 Foundry 注册的状态都会随时间变化：添加更多数据、构建应用程序、执行用户操作。随着 Foundry 产品的增长和演变，Linter 识别可节省时间、成本和资源的建议，以确保您的用例输出的交付方式针对您的目标进行了优化。

Linter 执行定期扫描以收集 Foundry 状态的响应式分析。扫描识别与预定义规则匹配的资源列表，并根据扫描结果生成建议列表。建议根据它们可能在您的 Foundry 注册中产生的影响进行排序。由于 Foundry 功能经常变化，Linter 扫描的结果是动态的，每天都会变化。

## 术语表

以下是帮助您浏览 Linter 的术语。

- **资源范围**：将根据扫描计划由 Linter 扫描的一组 Foundry 资源。资源范围由空间定义，规则针对空间中的资源运行。
- **规则范围**：将针对范围内的资源运行的一组规则。
- **扫描计划**：Linter 的配置，定义资源范围和规则范围。扫描计划必须属于空间，这是其默认资源范围。
- **扫描**：扫描计划的单次运行。扫描基于环境范围的计划运行。它们存储有关扫描的基本元数据，例如扫描状态、队列开始和结束时间，以及可能导致扫描失败的任何错误。
- **规则**：针对资源范围评估以生成一组建议的预定义逻辑集。规则随着 Foundry 中的产品和功能的变化而变化和演变。规则可以在预定义的规则预设中组合在一起，例如 `PIPELINE_COST_RULES` 预设。
- **建议**：与特定规则、资源和项目关联的建议。建议提供定义建议、受影响的资源以及如何应用建议的描述。
- **修复提案**：Linter 为实现建议而创建的提案。修复提案不会对底层资源进行更改，因此创建它们是安全的，可以由用户接受和应用。修复提案的示例是可以合并的个人资料更改的拉取请求。

**核心功能：**

### 模式 (Modes)
- 成本优化
- 可靠性优化
- 性能优化
- 可维护性优化

### 规则 (Rules)
- 预定义规则集
- 自定义规则配置
- 规则预设
- 影响跟踪

### 扫描 (Sweeps)
- 定期扫描
- 按需扫描
- 扫描调度
- 扫描历史

### 建议 (Recommendations)
- 优先级排序
- 影响估算
- 修复提案
- 实施指导
