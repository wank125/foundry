# HyperAuto (SDDI) 概览

> 更新时间：2025-12-28
> 文档来源：https://palantir.com/docs/foundry/hyperauto/overview/

## 概述

**HyperAuto** —— 也称为**软件定义数据集成（Software-Defined Data Integration，SDDI）** —— 是一套能力，旨在在组织中最常用和关键的系统之上提供开箱即用的端到端数据集成功能。

HyperAuto 是数字化转型的快速通道，使用户能够使用 ERP、CRM 和其他组织关键数据自主创建有价值的工作流。

## HyperAuto V2

### 新功能

**HyperAuto V2** 带来了新的用户体验和更高的性能，包括：

- **实时流支持**：支持实时数据流处理
- **更高性能**：提升数据处理性能
- **更快的数据准备**：几分钟内即可获得应用就绪的数据

### 支持的数据源

目前，HyperAuto V2 仅支持 **SAP 数据源**。

## 核心价值

### 主要功能

| 功能 | 说明 |
|------|------|
| **快速集成** | 在几分钟内同步、集成和构建数据 |
| **自动管道** | 自动创建数据管道 |
| **开箱即用** | 无需大量自定义开发 |
| **应用就绪** | 立即可用于应用构建 |

## HyperAuto V1

### 功能

HyperAuto V1 支持广泛的 ERP 和 CRM 源系统，包含以下综合功能：

- **派生元素（Derived Elements）**：通过内置智能提供
- **自动本体创建**：自动化创建 Ontology

### 可用性

- V1 将继续可用并得到支持
- 直到 V2 中提供所有源类型和必要功能
- V1 的更新将提前通知

## 架构

### 核心组件

HyperAuto V2 围绕数据集成工作流的三个主要组件提供编排和自动化：

```
┌─────────────────────────────────────────────────────────────┐
│              HyperAuto V2 架构                               │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  ┌─────────────────────────────────────────────────────┐   │
│  │  HyperAuto 编排层                                    │   │
│  │  ├── 自动化工作流                                    │   │
│  │  ├── 提案管理                                        │   │
│  │  └── 配置选项                                        │   │
│  └─────────────────────────────────────────────────────┘   │
│                          ↓                                  │
│  ┌─────────────┐  ┌─────────────────┐  ┌──────────────┐    │
│  │ Data Syncs  │→ │ Builder         │→ │ Ontology     │    │
│  │             │  │ Pipelines       │  │              │    │
│  │ • 源连接    │  │ • 自动生成      │  │ • 对象类型   │    │
│  │ • 实时流    │  │ • 数据转换      │  │ • 链接类型   │    │
│  │ • 增量同步  │  │ • 数据清洗      │  │ • Actions    │    │
│  └─────────────┘  └─────────────────┘  └──────────────┘    │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 1. Data Syncs

**功能**：
- 连接到数据源
- 配置同步设置
- 支持实时流

### 2. Builder Pipelines

**功能**：
- 自动生成管道
- 数据转换和清洗
- 构建数据集

### 3. Ontology

**功能**：
- 自动创建对象类型
- 创建链接类型
- 集成到应用

## 使用场景

### 1. ERP 数据集成

**场景**：快速集成 SAP ERP 数据。

**价值**：
- 几分钟内同步数据
- 自动构建数据管道
- 立即可用于分析

### 2. CRM 数据集成

**场景**：集成 Salesforce 或 NetSuite CRM 数据。

**价值**：
- 统一客户视图
- 实时数据同步
- 自动化工作流

### 3. 实时数据处理

**场景**：使用 HyperAuto V2 实时流处理数据。

**价值**：
- 实时数据可见性
- 快速决策支持
- 即时响应变化

## 配置选项

### HyperAuto V2 配置

配置过程包括以下步骤：

1. **名称和位置**：配置项目名称和位置
2. **数据源选择**：选择要连接的数据源
3. **同步设置**：配置数据同步设置
4. **管道配置**：配置构建管道选项
5. **本体设置**：配置本体集成选项
6. **提案管理**：通过提案管理变更

### 文件夹管道

- **SAP 文件夹管道**：支持基于文件夹的 SAP 管道
- **组织结构**：按文件夹组织管道
- **批量操作**：批量管理管道
